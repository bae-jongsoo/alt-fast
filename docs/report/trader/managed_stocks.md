# 현재 관리 종목 분석

## 1. 종목 관리 체계

### 시스템 구조
- **테이블**: `target_stocks` (is_active=True인 종목만 매매 대상)
- **종목 추가/삭제**: 웹 UI 또는 API (`POST /api/settings/stocks`, `DELETE /api/settings/stocks/{code}`)
- **Soft Delete**: 삭제 시 is_active=False로 변경 (이력 보존)
- **DART 연동**: dart_corp_code 필드로 공시 자동 수집

### 현재 종목 확인 방법
종목은 DB에 동적으로 관리되므로, 실제 목록은 다음으로 확인:
```bash
# API 조회
curl http://localhost:8000/api/settings/stocks

# 또는 DB 직접 조회
SELECT stock_code, stock_name, is_active FROM target_stocks WHERE is_active = true;
```

> **참고**: 코드베이스에서 확인 가능한 종목 예시 — 테스트 코드에서 삼성전자(005930), SK하이닉스(000660) 사용

## 2. 종목 선정 기준 (트레이더 관점)

단타매매에 적합한 종목 선정 기준은 다음과 같다:

### 2.1 유동성 (가장 중요)
- **일평균 거래대금 100억원 이상** 권장
- 거래량이 부족하면 원하는 가격에 매수/매도 불가 (슬리피지)
- 호가 스프레드가 넓어져 수수료 외 추가 비용 발생

### 2.2 변동성
- **일평균 변동폭(고가-저가) 2% 이상** 권장
- 변동성이 없으면 수수료+세금(0.23%) 이상의 수익 확보 어려움
- ATR 기준으로 종목별 변동성 순위를 관리하면 좋음

### 2.3 호가 단위 대비 가격
- 가격이 너무 높으면 (50만원+) 1틱 = 500원 이상 → 리스크 큼
- 가격이 너무 낮으면 (1,000원 미만) 1틱 = 1원 → 수수료 대비 비효율
- **적정 가격대**: 5,000원~200,000원 (코스닥 기준)

### 2.4 섹터 분산
- 동일 섹터 종목만 관리하면 시장 이벤트 시 전부 동시 하락 위험
- 반도체, 바이오, 2차전지, IT, 금융 등 섹터 분산 권장

### 2.5 시가총액
- **중대형주 (시총 5,000억 이상)**: 안정적 유동성, 낮은 변동성
- **중소형주 (시총 1,000~5,000억)**: 적절한 변동성 + 유동성
- **소형주 (시총 1,000억 미만)**: 높은 변동성이지만 호가 얇음, 세력 리스크

## 3. 종목별 보고서 분석 포인트

관리 종목 각각에 대해 보고서에서 다음을 추적해야 한다:

### 3.1 종목별 성과 카드

```
[종목 성과: 삼성전자 (005930)]
─────────────────────────
당일 매매: 2건 (승 1, 패 1)
누적 매매: 47건 (승 28, 패 15, 무 4)
──────────────────────────
당일 수익: -12,350원 (세후)
누적 수익: +234,500원 (세후)
──────────────────────────
당일 변동폭: 72,800 ~ 74,200 (1.92%)
수익 캡처율: 0.31 (캡처한 수익 / 변동폭)
──────────────────────────
평균 보유시간: 23분
최적 보유시간: 15분 (수익 건 평균)
──────────────────────────
호가 특성:
- 평균 스프레드: 0.07%
- 평균 체결강도: 112%
- 호가 두께: 상 (대형주)
```

### 3.2 종목 간 비교 지표

| 지표 | 설명 | 보고서 활용 |
|------|------|-----------|
| 승률 | 종목별 승리 비율 | 낮은 승률 종목 관리 대상에서 제외 검토 |
| 수익 캡처율 | 변동폭 대비 실현 수익 | 변동성은 큰데 캡처율 낮으면 전략 부적합 |
| 평균 보유시간 | 매수→매도 평균 시간 | 비정상적으로 긴 보유 = 판단 지연 |
| 반복매매 빈도 | 같은 종목 일일 매매 횟수 | 3회 이상 시 경고 |
| 일평균 거래대금 | 유동성 지표 | 감소 추세면 교체 검토 |

### 3.3 종목 교체 시그널

다음 조건 중 2개 이상 해당 시 종목 교체 권장:
1. 최근 5일 승률 30% 미만
2. 수익 캡처율 0.1 미만 (변동성 대비 수익 저조)
3. 일평균 거래대금이 50억원 미만으로 하락
4. 연속 3일 이상 손실

## 4. 9종목 포트폴리오 관점

### 동시 1종목 보유의 의미
- 현 시스템은 9개 종목을 관리하지만 **동시 보유는 1종목**
- 매매 사이클마다 9개 중 최적 1개를 선택하여 진입
- 나머지 8개는 "감시 대상" — 데이터만 수집

### 종목 선택의 중요성
- LLM이 9개 종목의 분봉/호가/뉴스/공시/재무를 모두 분석한 후 1개 선택
- **보고서 검증 필요**: 선택된 종목 vs 선택되지 않은 종목의 당일 성과 비교
  - "선택하지 않은 종목 중 더 나은 성과를 낸 종목이 있었는가?"
  - 이것이 보고서 항목 4번(HOLD 판단 복기)의 핵심

### 9종목 구성 최적화 방향
1. **섹터 커버리지**: 최소 3개 이상 섹터 분포
2. **변동성 밸런스**: 고변동 3 + 중변동 3 + 저변동 3
3. **시가총액 분산**: 대형 3 + 중형 3 + 소형 3
4. **정기 리밸런싱**: 월 1회 성과 기반 종목 교체
