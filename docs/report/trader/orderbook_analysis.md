# 호가창 분석법 — 실전 트레이딩 관점

## 1. 호가창 기본 구조

### 한국 주식시장 호가 체계
- **매도호가 (Ask)**: 팔고자 하는 가격과 수량. 위에서 아래로 낮아짐
- **매수호가 (Bid)**: 사고자 하는 가격과 수량. 위에서 아래로 낮아짐
- **스프레드**: 매도1호가 - 매수1호가. 좁을수록 유동성이 높음
- **현 시스템**: 5단계 호가 저장 (ask_price1~5, bid_price1~5 + 각 volume)

### 호가 단위 (코스피/코스닥)

| 가격대 | 호가 단위 (코스피) | 호가 단위 (코스닥) |
|--------|-------------------|-------------------|
| ~1,000원 | 1원 | 1원 |
| ~5,000원 | 5원 | 5원 |
| ~10,000원 | 10원 | 10원 |
| ~50,000원 | 50원 | 50원 |
| ~100,000원 | 100원 | 100원 |
| ~500,000원 | 500원 | 500원 |
| 500,000원~ | 1,000원 | 1,000원 |

호가 단위가 클수록 1틱 움직임의 수익/손실 영향이 큼.

## 2. 핵심 분석 지표

### 2.1 매수/매도 잔량 비율

```
매수비율 = total_bid_volume / (total_bid_volume + total_ask_volume) × 100
```

| 매수비율 | 해석 |
|---------|------|
| 70% 이상 | 강한 매수 수급. 상승 가능성 높음 |
| 55~70% | 약한 매수 우위. 추가 확인 필요 |
| 45~55% | 균형 상태. 방향성 불명확 |
| 30~45% | 약한 매도 우위 |
| 30% 이하 | 강한 매도 수급. 하락 가능성 높음 |

**주의**: 단순 잔량 비율만으로 판단하면 허매수/허매도에 속을 수 있음.

### 2.2 체결강도

```
체결강도 = (매수호가 체결량 / 매도호가 체결량) × 100
```

- **체결강도 > 100%**: 매수 체결이 매도 체결보다 많음 → 상승 압력
- **체결강도 < 100%**: 매도 체결이 매수 체결보다 많음 → 하락 압력
- **체결강도 > 150%**: 매우 강한 매수세
- **체결강도 < 70%**: 매우 강한 매도세

**보고서 적용**: 매매 시점의 체결강도와 매매 성과의 상관관계를 분석.
체결강도 120% 이상에서 매수한 건 vs 80% 이하에서 매수한 건의 승률 비교.

### 2.3 호가 두께 (Depth)

```
호가두께 = 5단계 총 잔량
얇은 호가 = 평소 대비 잔량이 50% 미만
```

- **얇은 호가**: 소량 주문으로도 가격 급변동 가능 → 슬리피지 위험
- **두꺼운 호가**: 안정적이지만 큰 움직임 기대 어려움

## 3. 매수벽과 매도벽

### 매수벽 (Bid Wall)
- 특정 매수호가에 비정상적으로 큰 잔량이 쌓인 상태
- **지지선 역할**: "이 가격 아래로는 내려가지 않겠다"는 심리적 방어선
- **판단 기준**: 해당 호가 잔량이 인접 호가 대비 3배 이상

### 매도벽 (Ask Wall)
- 특정 매도호가에 비정상적으로 큰 잔량이 쌓인 상태
- **저항선 역할**: "이 가격 위로는 올라가기 어렵다"는 심리적 저항
- **돌파 시 의미**: 매도벽이 체결로 소화되면 강한 상승 신호

### 매수벽/매도벽의 실전 해석

| 상황 | 해석 | 매매 판단 |
|------|------|---------|
| 매수벽 + 주가 하락 | 매수벽이 지지 중 | 관망 or 매수 검토 |
| 매수벽 소멸 + 주가 하락 | 지지 실패, 추가 하락 가능 | 손절 |
| 매도벽 + 체결 증가 | 매도벽 돌파 시도 | 돌파 시 매수 |
| 매도벽 + 주가 정체 | 저항 유효 | 관망 |

## 4. 허매수/허매도 (Spoofing) 판별

### 허매수 (Fake Bid)
- **목적**: 매수세가 강한 것처럼 보이게 하여 **자신의 보유 물량을 높은 가격에 매도**
- **특징**:
  - 큰 매수 잔량이 갑자기 출현
  - 실제 체결은 거의 없음 (잔량이 줄지 않음)
  - 주가가 오르면 매수 잔량이 사라짐 (주문 취소)

### 허매도 (Fake Ask)
- **목적**: 매도세가 강한 것처럼 보이게 하여 **낮은 가격에 매수(매집)**
- **특징**:
  - 큰 매도 잔량이 갑자기 출현
  - 체결강도는 오히려 매수 우위 (실제 매수가 진행 중)
  - 주가가 내리면 매도 잔량이 사라짐

### 판별 방법 (시스템 적용 가능)

```python
# 허매수/허매도 감지 로직 (개념)
def detect_spoofing(snapshots: list[OrderbookSnapshot]) -> str:
    """연속 스냅샷을 비교하여 허수 주문 감지"""

    prev, curr = snapshots[-2], snapshots[-1]

    # 잔량 급변 감지 (1분 사이 50% 이상 변동)
    bid_change = (curr.total_bid_volume - prev.total_bid_volume) / prev.total_bid_volume
    ask_change = (curr.total_ask_volume - prev.total_ask_volume) / prev.total_ask_volume

    # 잔량은 늘었는데 체결은 반대 방향
    if bid_change > 0.5 and actual_trades_are_sells:
        return "SUSPECTED_FAKE_BID"
    if ask_change > 0.5 and actual_trades_are_buys:
        return "SUSPECTED_FAKE_ASK"
```

## 5. 현 시스템의 호가 데이터 활용 현황

### 수집 구조
```
KIS WebSocket → Redis (ws:quote:{stock_code})
                  ↓
              trader.py:_save_orderbook_snapshot()
                  ↓
              orderbook_snapshots 테이블 (1분 1row)
```

### 프롬프트 투입 방식
- `build_buy_prompt()` / `build_sell_prompt()`에서 Redis의 최신 호가를 조회
- `asks`, `bids`, `total_ask_volume`, `total_bid_volume`을 JSON으로 LLM에 전달
- LLM이 호가 데이터를 얼마나 효과적으로 해석하는지가 핵심

### 보고서에서 검증해야 할 질문들

1. **호가 수급과 매매 성과의 상관관계**
   - 매수 시점 bid_volume > ask_volume 인 건의 승률 vs 반대 경우
   - 상관관계가 유의미하면 LLM 프롬프트에서 호가 가중치를 높여야 함

2. **호가 두께와 슬리피지**
   - 호가 잔량이 얇은 시점 매매의 실제 체결 가격 차이
   - 현재 가상매매이므로 직접 측정은 불가. 다만 호가 스프레드 크기로 간접 추정

3. **매수벽/매도벽과 보유 중 가격 변동**
   - 보유 중 상위 매도호가에 벽이 있을 때의 고점 캡처율
   - 하위 매수호가에 벽이 있을 때의 하방 방어 효과

4. **시간대별 호가 패턴**
   - 장초반 vs 점심 vs 장마감의 호가 두께 변화
   - 얇은 호가 시간대에서의 매매 성과

## 6. 보고서 항목 11번(호가 수급 분석) 구체화 제안

### 분석 지표

| 지표 | 계산 방식 | 의미 |
|------|---------|------|
| 매수비율 | bid_vol / (bid_vol + ask_vol) | 매매 시점 수급 상태 |
| 호가 비대칭도 | (bid_vol - ask_vol) / (bid_vol + ask_vol) | -1~1 범위, 양수=매수 우위 |
| 스프레드 비율 | (ask_price1 - bid_price1) / bid_price1 × 100 | 유동성 측정 |
| 1호가 집중도 | price1_vol / total_vol | 1호가에 잔량 몰림 정도 |
| 벽 존재 여부 | max(vol_i) / avg(vol) > 3 | 매수벽/매도벽 감지 |

### 보고서 출력 예시

```
[호가 수급 분석]
- 매수 수급 우위(bid>60%)에서 진입: 7건, 승률 71.4%
- 매도 수급 우위(ask>60%)에서 진입: 3건, 승률 33.3%
- 균형 상태에서 진입: 2건, 승률 50.0%
→ 수급 우위 확인 후 진입한 건의 승률이 유의미하게 높음

- 평균 스프레드: 0.08% (정상 범위)
- 스프레드 최대: 0.25% (11:47, 점심시간대 매매)
→ 점심시간대 호가 유동성 저하 확인
```
