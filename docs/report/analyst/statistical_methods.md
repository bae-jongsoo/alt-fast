## 개요

매매 성과 데이터를 통계적으로 분석할 때 필요한 방법론을 정리한다.

## 1. 소수 샘플 문제 (Small Sample Problem)

### 문제
- 일일 매매 건수가 5~20건 수준인 자동매매 시스템에서, 단일 일자의 승률/손익비는 **통계적 유의성이 거의 없음**
- 5건 중 4건 이김 → 승률 80%? → 다음날 1건 이기고 4건 지면 → 승률 20%
- 이런 변동은 순수 확률적 노이즈

### 대응 방법

#### (1) 롤링 윈도우 분석
```
롤링 승률(N) = 최근 N건의 거래 중 이익 거래 비율

권장 N: 30, 50, 100
```
- 일별 승률 대신 **최근 50거래 롤링 승률**을 추적
- 변동이 줄어들고 추세 파악이 가능

#### (2) 이항분포 신뢰구간 (Binomial Confidence Interval)
```
95% 신뢰구간 = p ± 1.96 × √(p(1-p)/n)

p = 관측된 승률
n = 거래 수
```

예시: 20건 중 12건 이김 (승률 60%)
```
신뢰구간 = 0.60 ± 1.96 × √(0.60 × 0.40 / 20)
         = 0.60 ± 0.215
         = [38.5%, 81.5%]
```
→ "실제 승률이 38%~82% 사이일 확률이 95%"라는 의미로, **20건으로는 결론 내리기 이르다**

#### (3) 누적 거래 기반 보고
- 일일 보고서에도 **누적 지표**를 함께 표시
- "오늘 승률 70% (5/7건) | 누적 승률 58% (누적 145/250건)"
- 누적 지표의 신뢰구간이 좁아질수록 전략 평가가 신뢰할 만해짐

### 의사결정 연결
- 일별 지표만 보고 전략을 바꾸면 과적합 위험
- **최소 50~100건의 거래가 쌓인 후에 전략 변경 판단**
- 일일 보고서에는 "오늘 숫자 + 누적 숫자 + 신뢰구간" 세 가지를 함께 표시

## 2. 연속 손실/이익 분석 (Streak Analysis)

### 확률 공식
```
연속 N회 손실 확률 = (패률)^N

예시: 패률 40%일 때
- 3연패 확률: 0.4³ = 6.4%
- 5연패 확률: 0.4⁵ = 1.02%
- 7연패 확률: 0.4⁷ = 0.16%
```

### 기대 최대 연패 길이
```
T건의 거래에서 기대 최대 연패 ≈ log(T) / log(1/패률)

예시: 승률 60%, 100건 거래
기대 최대 연패 ≈ log(100) / log(1/0.4) ≈ 2 / 0.398 ≈ 5.0건
```

### 실전 기대치 (승률별, 100건 거래 기준)

| 승률 | 기대 최대 연패 | 비고 |
|------|----------------|------|
| 50% | 6~7건 | 동전 던지기 수준 |
| 55% | 5~6건 | |
| 60% | 4~5건 | |
| 65% | 3~4건 | |
| 70% | 3건 | |

### 분석 포인트
1. **실제 연패가 기대치를 크게 초과하는가?**
   - 초과 시: 거래 간 독립성 가정 위반 가능 (시장 레짐 변화, 특정 시간대 집중 등)
2. **연패 후 행동 패턴**
   - 연패 후 포지션 축소 → 정상 (리스크 관리)
   - 연패 후 포지션 확대 → 위험 (복수매매 패턴)
3. **연승 분석도 동일하게 수행**
   - 비정상적 연승은 과도한 리스크 테이킹의 신호일 수 있음

### 의사결정 연결
- **report_plan.md 9번 항목(동일종목 반복매매)과 연결**
- 연패 구간에서 같은 종목을 반복 매매하는 패턴이 있는지 교차 분석
- 연패 길이 한도 설정 가능 (예: 5연패 시 당일 매매 중단 트리거)

## 3. 시간대별 성과 비교

### 분석 방법
```
각 시간대(시간 단위) 별:
- 매매 건수
- 승률
- 평균 수익률
- 총 손익
- 기대값(Expectancy)
```

### 통계적 검정: 카이제곱 검정 (Chi-Square Test)
시간대별 승률 차이가 통계적으로 유의한지 검정:

```
H0: 모든 시간대의 승률이 동일하다
H1: 적어도 한 시간대의 승률이 다르다

χ² = Σ (관측값 - 기대값)² / 기대값
```

### 주의사항
- 시간대별 매매 건수 차이가 크면 비교가 불공정
- **장 초반(9:00~9:30)**은 변동성이 구조적으로 크므로 별도 취급
- 시간대별 차이가 지속적인지 vs 특정 날만의 현상인지 구분 필요

### 의사결정 연결
- **report_plan.md 3번 항목(시간대별 수익 분석)의 통계적 프레임워크**
- 특정 시간대에 지속적으로 손실이면 → 해당 시간대 매매 제한 검토
- 시간대별 기대값이 양수인 구간만 매매하는 "시간 필터" 전략 가능

## 4. 종목별 성과 비교

### 분석 방법
```
각 종목별:
- 매매 횟수
- 승률, 손익비, 기대값
- 평균 보유 시간
- 총 손익
```

### 통계적 검정
- 종목 간 수익률 차이: **Kruskal-Wallis 검정** (비모수, 정규성 가정 불필요)
- 특정 종목이 전체 평균과 유의하게 다른지: **Mann-Whitney U 검정**

### 주의사항
- 종목별 매매 건수가 불균등하면 비교가 어려움
- 같은 종목이라도 시장 상황에 따라 성과가 달라짐
- **종목 자체의 특성**(변동성, 유동성)과 **전략의 적합성**을 분리해야 함

### 의사결정 연결
- 특정 종목에서 지속적으로 손실이면 → 해당 종목 제외 검토
- 변동성 높은 종목에서 성과가 좋다면 → 종목 선정 기준에 변동성 조건 추가

## 5. 매도 타이밍 분석 (Exit Quality)

### 고점 캡처율 (Peak Capture Rate)
```
고점 캡처율 = (매도가 - 매수가) / (보유 중 최고가 - 매수가) × 100
```

### 해석
| 캡처율 | 의미 |
|--------|------|
| > 80% | 매도 타이밍 우수 |
| 50~80% | 보통 |
| < 50% | 매도 타이밍 개선 필요 |
| < 0% | 고점 대비 손실 구간에서 매도 |

### 저점 회피율 (Trough Avoidance Rate)
```
저점 회피율 = (매수가 - 보유 중 최저가) / (매수가) × 100
```
- 이 값이 크면 매수 직후 가격이 크게 하락했다는 의미 → 매수 타이밍 점검

### 의사결정 연결
- **report_plan.md 2번 항목(놓친 기회 분석)의 핵심 정량 지표**
- 캡처율이 지속적으로 낮으면 → LLM 매도 프롬프트의 수익실현 기준 조정

## 6. 수익 분포 분석

### 히스토그램 / 분포 통계
```
- 왜도(Skewness): 양수면 오른쪽 꼬리(큰 이익 가끔), 음수면 왼쪽 꼬리(큰 손실 가끔)
- 첨도(Kurtosis): 높을수록 극단값 발생 빈도 높음
- 중앙값 vs 평균: 차이가 크면 극단값에 의한 왜곡
```

### 해석
- **양의 왜도(Positive Skew)**: 대부분 소폭 손실, 가끔 대폭 이익 → 추세추종 전략의 전형
- **음의 왜도(Negative Skew)**: 대부분 소폭 이익, 가끔 대폭 손실 → 옵션 매도형, 위험
- **높은 첨도**: "블랙스완" 리스크, 평균/표준편차만으로 리스크 과소평가 가능

### 의사결정 연결
- 음의 왜도 + 높은 첨도 = 가장 위험한 조합 (평소 잘 벌다가 한 번에 날림)
- 수익 분포의 모양이 전략의 성격을 보여줌

## 7. HOLD 판단 평가 프레임워크

### 혼동 행렬 (Confusion Matrix) 접근

|  | 실제 상승 | 실제 횡보/하락 |
|--|-----------|----------------|
| **매수 판단** | True Positive (정확한 매수) | False Positive (잘못된 매수) |
| **HOLD 판단** | False Negative (놓친 기회) | True Negative (올바른 관망) |

### 평가 지표
```
정밀도(Precision) = TP / (TP + FP) → 매수 중 실제로 오른 비율
재현율(Recall) = TP / (TP + FN) → 상승한 것 중 매수한 비율
F1 Score = 2 × (Precision × Recall) / (Precision + Recall)
```

### 의사결정 연결
- **report_plan.md 4번 항목(HOLD 판단 복기)의 정량적 프레임워크**
- Precision이 낮으면 → LLM이 너무 공격적 (사면 안 되는 것도 삼)
- Recall이 낮으면 → LLM이 너무 보수적 (기회를 놓침)
- F1으로 균형 추적

## 8. 통계적 유의성 검정 가이드

### 언제 검정이 필요한가?

| 상황 | 필요한 검정 | 최소 샘플 |
|------|------------|-----------|
| "이 전략은 수익이 나는가?" | 단일표본 t-검정 (기대값 > 0?) | 30건+ |
| "오전이 오후보다 좋은가?" | 독립표본 t-검정 또는 Mann-Whitney | 각 30건+ |
| "시간대별 승률이 다른가?" | 카이제곱 검정 | 각 셀 5건+ |
| "전략 변경 후 좋아졌나?" | 대응표본 t-검정 또는 부트스트래핑 | 30건+ |

### 일일 보고서에서의 접근
- 일일 데이터만으로는 대부분의 검정이 불가능 (샘플 부족)
- **누적 데이터 기반 검정 결과를 주간/월간 보고서에 포함**
- 일일 보고서에서는 "추세" 관찰에 집중, 통계적 결론은 누적 후 판단

### 부트스트래핑 (Bootstrap)
- 소수 샘플에서 신뢰구간을 추정하는 비모수 방법
- 원본 데이터에서 복원추출(resampling)을 10,000회 수행하여 분포 추정
- 정규성 가정이 불필요하므로 트레이딩 데이터에 적합

```python
# 부트스트래핑 예시 (개념)
import numpy as np

returns = [0.5, -0.3, 1.2, -0.8, 0.4, ...]  # 거래별 수익률
n_bootstrap = 10000
means = []
for _ in range(n_bootstrap):
    sample = np.random.choice(returns, size=len(returns), replace=True)
    means.append(np.mean(sample))

ci_lower = np.percentile(means, 2.5)
ci_upper = np.percentile(means, 97.5)
```

### 의사결정 연결
- p-value < 0.05가 아니면 "아직 판단 보류"로 보고
- 절대 소수 샘플로 확정적 결론 내리지 않기
- "통계적으로 유의하지 않음 ≠ 효과 없음", 단지 아직 증거가 부족한 것
