# 일일 보고서 설계 모범 사례

## 개요

프로 트레이딩 팀과 헤지펀드의 보고서 설계 원칙을 기반으로,
일일 상세 트레이딩 리포트의 구성, 정보 계층, 시각화 방법을 정리한다.

---

## 1. 보고서 정보 계층 (Information Hierarchy)

### 원칙: 역삼각형 구조
가장 중요한 정보를 먼저, 상세는 뒤로.

```
[Level 1] 핵심 숫자 — 5초 안에 파악
[Level 2] 주요 분석 — 30초 안에 이해
[Level 3] 상세 데이터 — 필요할 때 참조
```

### 권장 구성 순서

#### Level 1: Executive Summary (핵심 대시보드)
| 항목 | 표시 예시 |
|------|-----------|
| 당일 순손익 | +127,500원 (세후) |
| 수익률 | +0.85% |
| 벤치마크 대비 | 코스피 -0.32% → Alpha +1.17% |
| 매매 횟수 | 12건 (승 8 / 패 4) |
| 승률 | 66.7% |
| Profit Factor | 2.14 |
| 일중 MDD | -0.42% |

> **설계 의도:** 보고서를 열자마자 "오늘 잘했나 못했나"를 즉시 판단할 수 있어야 함.
> 헤지펀드 일일 보고서에서 가장 먼저 나오는 것은 항상 P&L(손익)과 벤치마크 대비 성과.

#### Level 2: 주요 분석
1. **자산 변동 곡선 (Equity Curve)** — 장중 자산 변화 시각화
2. **종목별 매매 타임라인** — 오늘 뭘 했는지 팩트
3. **시간대별 수익 분석** — 언제 벌고 언제 잃었나
4. **놓친 기회 분석** — 매도 타이밍 품질
5. **HOLD 판단 복기** — 안 산 것도 평가

#### Level 3: 상세/참고
6. 변동성 대비 성과
7. LLM 판단근거 복기
8. 동일종목 반복매매 패턴
9. 호가 수급 분석 (데이터 축적 후)
10. 누적 통계 (롤링 승률, 기대값 추이)

## 2. 시각화 가이드

### 2.1 Equity Curve (자산 변동 곡선)

**가장 중요한 차트.** 모든 프로 보고서의 첫 번째 시각화.

#### 구성 요소
```
[상단 패널] 시계열 자산 가치 (라인 차트)
  - X축: 시간 (9:00~15:30)
  - Y축: 평가자산 (원)
  - 매수/매도 시점 마커 표시

[하단 패널] Drawdown 영역 (아래로 향하는 면적 차트, 빨간색)
  - X축: 시간 (상단과 동일)
  - Y축: 고점 대비 낙폭 (%)
```

#### 모범 사례
- 시작점을 0%로 정규화하면 수익률 비교에 용이
- 벤치마크(코스피) 곡선을 함께 그리면 Alpha 시각화
- Drawdown 패널을 함께 그려야 "중간에 얼마나 힘들었나" 파악 가능
- 매수(▲ 초록) / 매도(▼ 빨강) 마커로 매매 시점 표시

### 2.2 시간대별 수익 (Bar Chart)

```
X축: 시간대 (9시, 10시, ..., 15시)
Y축: 손익 (원)
색상: 양수 초록, 음수 빨강
막대 위에 매매 건수 표시
```

### 2.3 종목별 손익 (Horizontal Bar Chart)

```
Y축: 종목명
X축: 손익 (원)
정렬: 손익 큰 순서로
색상: 양수 초록, 음수 빨강
```

### 2.4 수익 분포 (Histogram)

```
X축: 거래별 수익률 (%)
Y축: 빈도
평균선, 중앙값선 표시
0% 기준선 강조
```

### 2.5 정보 과부하 방지 원칙

| 원칙 | 설명 |
|------|------|
| **3차트 규칙** | 핵심 차트는 3개 이하 (Equity Curve, 시간대별, 종목별) |
| **숫자 7±2 법칙** | 한 화면에 7개 이상의 독립 지표 표시 금지 |
| **비교 가능하게** | 절대값보다 비율/비교 형태 선호 |
| **색상 일관성** | 이익=초록, 손실=빨강, 전체에서 통일 |
| **점진적 공개** | 기본은 요약, 클릭하면 상세 (API/프론트 설계) |

## 3. 일일 vs 주간 vs 월간 보고서 차이

### 일일 보고서 (Daily)
| 특징 | 설명 |
|------|------|
| **목적** | 오늘의 매매 사후 분석, 즉각적 피드백 |
| **핵심 지표** | P&L, 승률, PF, MDD, Alpha |
| **통계적 가치** | 낮음 (샘플 부족) |
| **포함할 것** | 개별 거래 상세, 타임라인, 놓친 기회 |
| **포함하지 말 것** | 확정적 통계 결론, 장기 예측 |
| **톤** | "오늘 이런 일이 있었다" (기술적) |

### 주간 보고서 (Weekly)
| 특징 | 설명 |
|------|------|
| **목적** | 주간 추세 파악, 패턴 발견 |
| **핵심 지표** | 누적 승률, 기대값 추이, 시간대/종목별 성과 |
| **통계적 가치** | 중간 (50~100건이면 일부 검정 가능) |
| **포함할 것** | 롤링 지표, 연속 손실/이익 분석, 요일별 비교 |
| **포함하지 말 것** | 개별 거래 상세 (일일에 위임) |
| **톤** | "이번 주 어떤 패턴이 보인다" (분석적) |

### 월간 보고서 (Monthly)
| 특징 | 설명 |
|------|------|
| **목적** | 전략 성과 평가, 파라미터 조정 근거 |
| **핵심 지표** | Sharpe/Sortino, Alpha/Beta, Kelly %, 신뢰구간 |
| **통계적 가치** | 높음 (200~500건이면 대부분 검정 가능) |
| **포함할 것** | 통계적 유의성 검정, 전략 변경 전후 비교 |
| **포함하지 말 것** | 일별 상세 |
| **톤** | "전략을 이렇게 바꿔야 한다" (처방적) |

## 4. 텔레그램 요약 보고서 설계

### 4096자 제한 내 최적 구성

```
📊 [날짜] 일일 리포트

💰 손익: +127,500원 (세후)
📈 수익률: +0.85% | 코스피 대비 Alpha +1.17%
🎯 매매: 12건 (승8/패4, 승률 66.7%)
⚖️ PF: 2.14 | 기대값: +15,200원/건
📉 일중 MDD: -0.42%

── 종목별 ──
삼성전자: +52,000원 (2건)
SK하이닉스: +38,500원 (1건)
LG에너지: -12,000원 (1건)
...

── 시간대 ──
9시대: +45,000원 (5건)
10시대: +82,500원 (4건)
11시대: +0원 (0건)
14시대: -12,000원 (3건)

── 주요 이벤트 ──
• 최대 이익: SK하이닉스 +38,500원 (10:23~10:47)
• 최대 손실: LG에너지 -12,000원 (14:05~14:32)
• 놓친 기회: 삼성전자 고점 대비 72% 캡처

누적 승률: 58.2% (145/249건)
```

### 설계 원칙
1. **첫 5줄에 모든 핵심** → 알림만 보고도 판단 가능
2. **종목별은 손익 순 정렬** → 큰 영향 먼저
3. **시간대는 매매 있는 곳만** → 공백 시간 생략
4. **주요 이벤트는 3개 이하** → 정보 과부하 방지
5. **누적 지표 한 줄** → 맥락 제공

## 5. API 응답 구조 설계 가이드

### 계층적 JSON 구조

```json
{
  "date": "2026-03-26",
  "summary": {
    "pnl_net": 127500,
    "return_pct": 0.85,
    "alpha": 1.17,
    "trades_count": 12,
    "win_rate": 66.7,
    "profit_factor": 2.14,
    "expectancy": 15200,
    "intraday_mdd": -0.42
  },
  "equity_curve": [...],
  "trades": [...],
  "analysis": {
    "by_hour": [...],
    "by_stock": [...],
    "missed_opportunities": [...],
    "hold_review": [...],
    "repeated_trades": [...]
  },
  "cumulative": {
    "win_rate": 58.2,
    "total_trades": 249,
    "confidence_interval_95": [53.1, 63.3]
  }
}
```

### 설계 원칙
- `summary`는 항상 첫 번째 → 프론트엔드에서 먼저 렌더링
- `analysis` 내부는 독립적 → 필요한 것만 로딩 가능
- `cumulative`로 맥락 제공
- 텔레그램/프론트/CLI 모두 같은 API에서 필요한 부분만 사용

## 6. report_plan.md 11개 항목에 대한 분석가 보완 의견

### 보완이 필요한 항목

#### 5번 (승률/손익비)에 추가할 지표
- **기대값(Expectancy)**: 승률과 손익비를 통합하는 가장 중요한 지표인데 빠져 있음
- **수익 분포 통계**: 왜도/첨도를 추가하면 "평균"의 신뢰성을 판단 가능
- **신뢰구간**: 소수 샘플 문제를 명시적으로 다뤄야 함

#### 8번 (코스피/코스닥 대비)에 추가할 분석
- **Beta 계산**: 단순 수익률 차이 외에 시장 민감도(Beta) 계산 추가
- **Alpha의 통계적 유의성**: Alpha가 실력인지 운인지 판단하려면 누적 데이터 필요

#### 10번 (Equity Curve)에 추가할 지표
- **Recovery Time**: MDD에서 복구까지 걸린 시간
- **Underwater Curve**: 고점 대비 현재 낙폭의 시계열 (Drawdown 차트)

### 추가 제안 항목

#### A. 누적 추적 지표 섹션
- report_plan.md의 모든 분석이 "오늘" 기준인데, **누적 추적** 관점이 빠져 있음
- 롤링 승률, 롤링 기대값, 롤링 Sharpe 등의 시계열 추가 권장
- 일일 보고서에 "오늘 숫자 vs 누적 숫자" 비교 포함

#### B. 리스크 관리 지표 섹션
- 일중 최대 동시 보유 종목 수 / 금액
- 최대 단일 종목 노출 비율
- 현금 비중 시계열 (장중 현금 비율 변화)

#### C. 슬리피지 분석
- 지정가 대비 실제 체결가 차이 분석
- 시간대별/종목별 슬리피지 패턴
- 매매 비용(수수료 + 세금 + 슬리피지)이 수익을 얼마나 깎는지

### 중요도 평가

| 추가 항목 | 중요도 | 이유 |
|-----------|--------|------|
| 기대값 추가 (5번 보완) | ★★★ | 가장 중요한 단일 지표가 빠져 있음 |
| 누적 추적 섹션 (신규 A) | ★★★ | 일별만으로는 의사결정 불가 |
| 신뢰구간 표시 (5번 보완) | ★★☆ | 소수 샘플 과신 방지 |
| Underwater Curve (10번 보완) | ★★☆ | MDD 시각화에 필수 |
| 리스크 관리 지표 (신규 B) | ★★☆ | 리스크 모니터링 |
| 슬리피지 분석 (신규 C) | ★☆☆ | 중요하나 데이터 확보 필요 |
| Beta 계산 (8번 보완) | ★☆☆ | 데이 트레이딩에서는 의미 제한적 |
