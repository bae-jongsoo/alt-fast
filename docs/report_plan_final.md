# 일일 상세 트레이딩 리포트 — 최종 기획

## 개요

하루의 매매를 정량적으로 사후 분석하는 상세 보고서.
기존 `alt review daily`(LLM 텍스트 리뷰)와 별개로, 숫자 기반 팩트 시트를 제공한다.

### 설계 원칙
- **역삼각형 구조**: 가장 중요한 정보를 먼저, 상세는 뒤로 (5초 → 30초 → 필요시 참조)
- **일일 + 누적 병행**: 오늘 숫자만으로는 판단 불가, 누적 지표와 항상 함께 표시
- **신뢰구간 명시**: 소수 샘플(일 5~20건)에서 확정적 결론을 내리지 않음
- **세후 기준**: 모든 손익은 수수료+세금 반영 후 기준 (profit_loss_net)

### 시스템 특성 (보고서 해석의 전제)
- LLM 기반 데이트레이딩, 60초 주기 판단
- 동시 보유 1종목, 감시 대상 9종목
- 총 거래비용 ~0.23% (매수 0.015% + 매도 0.015% + 거래세 0.2%)
- 표면 손익비 1.5:1 이상이어야 실질 수익 가능

## 분석 항목 (13개)

### 1. 종목별 매매 타임라인

**뭘 보나:** 종목별로 몇시에 사서 몇시에 팔았는지, 보유시간, 손익
**왜 필요한가:** 가장 기본적인 "오늘 뭘 했는지" 팩트 시트

**포함 지표:**
- BUY-SELL 페어별: 매수/매도 시각, 체결가, 수량, 보유시간, 세후 손익
- 보유시간 구간 분류: ~10분(초단타), ~30분(단타), ~1시간(스윙단타), 1시간+(중기)
- 장 시간대 태그: 각 매매에 "장초반/오전장/점심/오후장/마감전/동시호가" 태그
- 진입 속도: LLM 판단(decision created_at) → 체결(result_executed_at) 지연시간

**데이터:** `order_histories` (buy_order_id FK로 BUY→SELL 매칭), `decision_histories` (판단 시각)

---

### 2. 놓친 기회 분석

**뭘 보나:** 매도 타이밍과 매수 타이밍의 품질을 정량적으로 측정
**왜 필요한가:** 고점 대비 몇 %를 캡처했는지 추적하면 매도 프롬프트 개선 근거

**포함 지표:**
- **고점 캡처율**: (매도가 - 매수가) / (보유 중 최고가 - 매수가) × 100
  - 80% 이상: 우수, 50~80%: 보통, 50% 미만: 개선 필요
- **보유 중 MDD**: 매수 후 최저가까지의 하락폭. 최종 수익이 있어도 중간 낙폭이 크면 위험 매매
- **저점 회피율**: (매수가 - 보유 중 최저가) / (당일고가 - 당일저가) → 매수 타이밍 품질
- **시간 기반 분석**: "N분만 더/덜 보유했으면" 최적 보유시간 추정

**데이터:** `minute_candles` (보유 구간의 고가/저가), `order_histories`

---

### 3. 시간대별 수익 분석

**뭘 보나:** 트레이더 시간대 6구간별 매매 횟수, 승률, 손익, 기대값
**왜 필요한가:** 시간대별 패턴 발견 → 매매 시간대 제한/집중 전략 근거

**6구간 분류:**

| 구간 | 시간 | 특성 |
|------|------|------|
| 장초반 | 09:00~09:30 | 골든타임, 높은 변동성 |
| 오전장 | 09:30~11:30 | 안정적 추세 형성 |
| 점심시간 | 11:30~13:00 | 낮은 거래량, 비효율 구간 |
| 오후장 | 13:00~14:30 | 기관/외인 방향성 |
| 마감접근 | 14:30~15:20 | 포지션 정리 |
| 동시호가 | 15:20~15:30 | 종가 결정 |

**경고 규칙:**
- 점심시간대 매매가 3건 이상이면 경고 (비효율 구간)
- 구간별 기대값이 음수이면 해당 구간 매매 재고 권고

**데이터:** `order_histories` (result_executed_at 기준 그룹핑)

### 4. HOLD 판단 복기

**뭘 보나:** LLM이 "관망" 판단한 뒤 30분/60분 후 해당 종목의 실제 가격 변동
**왜 필요한가:** "안 산 것"도 평가해야 전략의 절반이 보임

**판정 기준:**
- MISSED_UP: HOLD 후 일정 이상 상승 → 놓친 기회
- CORRECT_HOLD: 횡보 → 올바른 판단
- AVOIDED_DROP: 하락 → 잘 피한 것

**추가 분석:**
- **HOLD vs 선택된 종목 비교**: "A종목 HOLD → +2%, B종목 BUY → -0.5%" → 종목 선택 품질 측정
- **HOLD 빈도**: >70%이면 보수적, <20%이면 공격적 경향
- **혼동 행렬 관점**: Precision(매수 중 실제 상승 비율) / Recall(상승 중 매수한 비율) 추적
- **HOLD 시점 호가 상태**: HOLD 당시 수급이 매수/매도 우위였는지

**데이터:** `decision_histories` (HOLD 건) + `minute_candles`, `orderbook_snapshots`

### 5. 승률 / 손익비 / 기대값

**뭘 보나:** 전략의 수익성을 통합적으로 평가하는 핵심 지표 세트
**왜 필요한가:** 승률만으론 부족. 손익비, 기대값, Profit Factor를 함께 봐야 실질 수익성 판단 가능

**포함 지표:**

| 지표 | 공식 | 의미 |
|------|------|------|
| 승률 | 이익 거래 수 / 전체 × 100 | 기본 지표 |
| 평균 이익/손실 (세후) | 세후 기준 | 실질 금액 |
| 손익비 | 평균 이익 / \|평균 손실\| | 이기고 질 때의 비율 |
| **기대값 (Expectancy)** | (승률 × 평균이익) - (패률 × \|평균손실\|) | **가장 중요한 단일 지표** |
| Profit Factor | 총이익 / \|총손실\| | 1.5~2.5 목표 |
| 실질 손익비 | 수수료 반영 후 손익비 | 표면 vs 실질 괴리 확인 |
| 최대 연속 승/패 | 연속 승 N건 / 연속 패 N건 | 멘탈 리스크 |
| Kelly Criterion | (W×R - L) / R | 최적 투입 비율 → Half Kelly로 적용 |

**신뢰구간 표시 (필수):**
```
오늘 승률: 66.7% (8/12건)
누적 승률: 58.2% (145/249건) [95% CI: 52.0%~64.2%]
```

**데이터:** `order_histories` (SELL 건의 profit_loss_net)

### 6. 변동성 대비 성과

**뭘 보나:** 종목의 당일 변동폭 대비 실제 캡처한 수익의 비율
**왜 필요한가:** 절대 수익보다 "기회 대비 얼마나 먹었나"가 전략 품질의 진짜 지표

**포함 지표:**
- **캡처율**: 실현수익 / 일중 변동폭(고가-저가) × 100
- **ATR 기반 캡처율**: 실현수익 / ATR(14분봉) → 더 정밀한 측정
- **변동성-수익 관계**: 변동폭 구간별(저<1%, 중1~3%, 고>3%) 승률/수익
- **시간 효율비**: 실현수익 / 보유시간(분) → 분당 수익 효율

**데이터:** `minute_candles` (일중 고가/저가, ATR 계산) + `order_histories`

---

### 7. LLM 판단근거 복기

**뭘 보나:** LLM이 매매 판단 시 사용한 소스별 가중치와, 그 소스에 의존한 판단의 승패율
**왜 필요한가:** 소스별 적중률이 프롬프트 튜닝의 직접적 근거

**분석 방법:**
- `parsed_decision.decision.sources`의 type별(기술적분석, 뉴스, 공시, 수급) 가중치 추출
- 각 소스 type에 높은 가중치(>30%)를 부여한 판단들의 승패율 집계
- 호가 소스의 가중치 추적을 별도 강조 (호가 스냅샷 데이터와 교차 검증)

**데이터:** `decision_histories` (parsed_decision.sources) + `order_histories` (승패 결과)

---

### 8. 코스피/코스닥 대비 수익률

**뭘 보나:** 당일 시장 등락률 vs 시스템 수익률. Alpha(초과수익) 측정
**왜 필요한가:** 시장이 2% 오르는 날 1% 벌었으면 사실 언더퍼폼

**포함 지표:**
- **일간 Alpha**: 시스템 수익률 - 코스피 등락률
- **종목별 Alpha**: 해당 종목 당일 등락률 대비 해당 종목에서의 수익률
- **시장 상태 분류**: 상승장(>1%), 횡보(-1%~1%), 하락장(<-1%)별 시스템 성과
  - "하락장에서도 수익을 내는가?" → 진짜 알파 측정

**데이터 확보:** 보고서 생성 시점에 KIS API로 당일 코스피/코스닥 종가 조회 (별도 테이블 불필요)

### 9. 동일종목 반복매매 패턴

**뭘 보나:** 같은 종목을 하루에 여러 번 매매했을 때, 회차별 수익률 변화
**왜 필요한가:** LLM이 같은 종목을 반복 선택하면서 수익이 감소하는 "복수매매" 패턴 탐지

**포함 지표:**
- 회차별 수익률 변화: 1차, 2차, 3차... 수익률 추이
- **경고**: 동일종목 3회 이상 매매 시 경고 플래그
- **수수료 누적**: 반복매매의 수수료 총합 vs 수익 총합 ("반복하지 않았으면 +0.57%였는데...")
- **LLM 편향 감지**: 특정 종목 선택 빈도가 비정상적으로 높으면 프롬프트 편향 의심

**데이터:** `order_histories` (같은 stock_code의 복수 BUY-SELL 페어)

### 10. 자산 변동 곡선 (Equity Curve)

**뭘 보나:** 하루 동안의 평가자산 변화를 시계열로
**왜 필요한가:** 최종 수익이 +1%여도 중간에 -3%까지 갔다 돌아온 거라면 리스크가 큰 것

**포함 지표:**
- **Equity Curve**: 시간별 평가자산 (매수/매도 시점 마커 표시)
- **Underwater Curve**: 고점 대비 낙폭의 시계열 (빨간색 면적 차트)
- **일중 MDD**: 장중 최대 낙폭
- **회복 시간**: MDD 발생 후 원금 복귀까지 소요 시간
- **연속 하락 구간**: 자산이 연속 줄어든 최장 구간 (시간)
- **일일 한도 경고**: MDD가 설정 한도(예: -3%) 초과 시 경고

**시각화 (프론트엔드):**
```
[상단] 시계열 자산가치 라인차트 + 벤치마크(코스피) 곡선 + 매수(▲)/매도(▼) 마커
[하단] Drawdown 면적차트 (빨간색, 아래방향)
```

**데이터:** `minute_candles` + `order_histories` 조합으로 시뮬레이션 계산

---

### 11. 호가 수급 분석

**뭘 보나:** 매매 시점의 호가창 상태(매수잔량/매도잔량 비율)와 매매 성과의 관계
**왜 필요한가:** 유의미하면 LLM 판단 소스에 호가 정보 추가하여 판단 품질 향상 가능

**포함 지표:**
- 매수잔량 >> 매도잔량(수급 우위)일 때 vs 반대일 때 승률 비교
- 호가 잔량이 얇은 시점의 슬리피지 분석
- 체결강도(매수체결/매도체결) 연동 분석
- 1호가 집중도와 체결 속도의 관계
- 허매수/허매도 감지 (연속 스냅샷 비교로 급변 잔량 탐지)

**선행 조건:** `orderbook_snapshots` 데이터 축적 필요 (현재 수집 중)

**데이터:** `orderbook_snapshots` + `order_histories`

---

### 12. 매매 빈도 적정성 분석 (신규)

**뭘 보나:** 하루 총 매매 횟수, 판단 대비 실행 비율, 수수료 비중
**왜 필요한가:** 너무 잦은 매매 → 수수료 누적, 너무 적은 매매 → 기회 놓침. 적정 빈도 찾기가 시스템 최적화의 핵심

**포함 지표:**
- 총 판단 횟수 (BUY 판단 시도) vs 실제 매수 실행 (실행률)
- HOLD 판단 비율
- 시간당 평균 매매 건수
- **수수료 비중**: 수수료 총액 / 순이익 × 100
  - 30% 초과 시 경고: "매매 빈도를 줄이거나 목표 수익폭을 키우세요"

**데이터:** `decision_histories` + `order_histories`

---

### 13. 매수가 최적성 분석 (신규)

**뭘 보나:** 실제 매수가가 당일 가격 범위에서 어디에 위치하는지
**왜 필요한가:** "좋은 종목을 골랐지만 비싸게 산" 경우 식별. 매수 타이밍 품질 측정

**포함 지표:**
- **매수 위치**: (매수가 - 당일저가) / (당일고가 - 당일저가) × 100
  - 0% = 최저가에서 매수 (최적), 100% = 최고가에서 매수 (최악)
- 최적 매수 비율: 하위 30% 가격대에서 진입 성공한 비율
- 평균 매수 후 추가 하락폭: 진입 후 더 빠진 평균

**데이터:** `minute_candles` (당일 고/저가) + `order_histories` (매수 체결가)

## 구현 순서

| 단계 | 작업 | 의존성 | 항목 |
|------|------|--------|------|
| 1 | 보고서 서비스 코어 (핵심 지표) | 없음 | 1, 5, 10 |
| 2 | 보고서 서비스 확장 (분석 항목) | 단계 1 | 2, 3, 4, 6, 8, 9, 12, 13 |
| 3 | LLM/호가 분석 | 단계 1 | 7, 11 |
| 4 | API 엔드포인트 + CLI 커맨드 | 단계 1~3 | 전체 |
| 5 | 텔레그램 포맷 | 단계 4 | 전체 |
| 6 | 누적 추적 지표 | 단계 4, 데이터 축적 후 | 전체 |

### 서비스 구현 패턴 (Developer 권고)
- 기존 서비스 패턴 준수: `app/services/` 하위에 `report_service.py` 생성
- async/await + SQLAlchemy 2.x 패턴
- 분석 항목별 메서드 분리 → API에서 필요한 것만 호출 가능
- KIS API 호출은 기존 `kis_api.py` 패턴 활용
